Crie uma aplicação web completa em Python usando o framework FastAPI. A aplicação funcionará como um agente de IA para assessores financeiros, com as seguintes funcionalidades e estrutura:

**1. Objetivo Principal:**
O agente de IA se comunicará via WhatsApp, usando uma API HTTP como o WAHA para se conectar a um número de telefone. Ele responderá a perguntas usando uma busca semântica em uma base de conhecimento do Notion. A aplicação também terá um painel de administração com um sistema de chamados (ticketing) no estilo Kanban e gerenciamento de usuários.

**2. Estrutura de Pastas e Arquivos:**
Gere a seguinte estrutura de arquivos e pastas:
- `/main.py`: Ponto de entrada da aplicação FastAPI.
- `/indexer.py`: Um script autônomo para popular a base de dados vetorial a partir do Notion.
- `/requirements.txt`: Com todas as dependências necessárias.
- `/core/`: Para configurações e segurança.
  - `config.py`: Carrega variáveis de ambiente (secrets) como chaves de API.
  - `security.py`: Funções para hashing de senhas e gerenciamento de tokens JWT.
- `/database/`: Para o banco de dados relacional.
  - `database.py`: Configuração do SQLAlchemy para um banco de dados SQLite.
  - `models.py`: Modelos SQLAlchemy para as tabelas `User` e `Ticket`.
  - `crud.py`: Funções para criar, ler, atualizar e deletar usuários e tickets.
- `/services/`: Para a lógica de negócio e integrações.
  - `whatsapp_client.py`: Para enviar mensagens via API HTTP do WhatsApp.
  - `openai_agent.py`: Para gerar respostas usando a API da OpenAI.
  - `vector_store.py`: Para gerenciar a busca semântica no banco de dados vetorial ChromaDB.
- `/api/endpoints/`: Para os endpoints da API.
  - `whatsapp_webhook.py`: Recebe mensagens do WhatsApp.
  - `auth.py`: Para login e autenticação.
  - `users.py`: Para gerenciar usuários.
  - `tickets.py`: Para gerenciar os chamados (tickets).
- `/frontend/templates/`: Para as páginas HTML do painel de administração.
  - `login.html`: Página de login.
  - `kanban.html`: Quadro Kanban interativo para os chamados.
  - `admin.html`: Painel para gerenciamento de usuários.

**3. Detalhes Técnicos e Lógica:**

- **Dependências:** Use `fastapi`, `uvicorn`, `sqlalchemy`, `pydantic`, `openai`, `httpx`, `python-dotenv`, `passlib[bcrypt]`, `python-jose[cryptography]`, `jinja2`, `chromadb`, e `tiktoken`. Liste todas em `requirements.txt`.

- **Busca Semântica:**
  - O `indexer.py` deve ler páginas de um banco de dados do Notion, extrair o texto, gerar embeddings usando o modelo "text-embedding-3-small" da OpenAI e armazená-los no ChromaDB.
  - O `services/vector_store.py` deve ter uma função que recebe uma pergunta, gera seu embedding e consulta o ChromaDB para encontrar os textos mais relevantes.

- **Fluxo do Agente de IA:**
  1. Uma mensagem chega ao webhook em `whatsapp_webhook.py`.
  2. O sistema identifica o usuário pelo número de telefone.
  3. A pergunta do usuário é usada para fazer uma busca semântica no `vector_store.py`.
  4. A pergunta e o contexto encontrado são enviados para o `openai_agent.py`.
  5. A IA gera uma resposta. Se o contexto não for suficiente, ela deve ser instruída a perguntar se o usuário deseja abrir um chamado.
  6. Se o usuário confirmar a abertura do chamado, o sistema cria um `Ticket` no banco de dados e notifica o broker responsável.

- **Painel de Administração:**
  - Protegido por um sistema de login (`auth.py` ) que gera tokens JWT.
  - A página `kanban.html` deve usar JavaScript (pode ser com a biblioteca SortableJS) para permitir que os usuários arrastem os cards de tickets entre as colunas ("Aberto", "Em Andamento", "Concluído") e atualize o status no banco de dados através de uma chamada à API.
  - Apenas usuários com a role "admin" ou "broker" podem acessar o painel.

- **Configuração:** O `core/config.py` deve ser configurado para ler todas as chaves de API (OpenAI, Notion, WAHA) e a `SECRET_KEY` do JWT a partir das "Secrets" do Replit.

Por favor, gere o código completo para todos os arquivos, com comentários explicando as partes mais importantes da lógica.
