# Melhoria Incremental do Agente de WhatsApp (Zapi)

## Objetivo

Expandir o agente de WhatsApp para:

* Receber áudios
* Receber imagens
* Receber arquivos (PDF, DOC, etc)
* Enviar áudios
* Enviar imagens
* Enviar arquivos

Sem alterar fluxos existentes de texto.
A implementação deve ser **estritamente incremental**.

---

## Princípios Obrigatórios

* ❌ Não alterar lógica atual de mensagens de texto

* ❌ Não renomear variáveis, handlers ou estruturas existentes

* ❌ Não refatorar componentes atuais

* ❌ Não mudar banco de dados, schemas ou storage

* ❌ Não criar lógica de preview persistente

* ✅ Apenas adicionar novos fluxos paralelos

* ✅ Apenas adicionar novos handlers

* ✅ Apenas adicionar novos componentes visuais opcionais

* ✅ Usar a Zapi já conectada

* ✅ Usar React + Flowbite para UI

---

## Escopo Funcional

### Tipos de mídia suportados

Implementar suporte para:

* Áudio (voice / audio)
* Imagem (image)
* Arquivo genérico (document / file)

Cada tipo deve ter:

* Recebimento via webhook/evento da Zapi
* Identificação do tipo de mensagem
* Tratamento incremental
* Renderização básica no frontend
* Envio via Zapi

---

## Backend / API (Incremental)

### 1. Identificação de tipo de mensagem

Adicionar camada de detecção de tipo:

* Texto
* Áudio
* Imagem
* Documento/Arquivo

Sem mudar o fluxo atual de texto.

Apenas:

* Criar ramificações adicionais para tipos não-texto
* Garantir que texto continue funcionando como hoje

---

### 2. Recebimento de Áudio

Adicionar lógica incremental para:

* Detectar mensagens de áudio no payload da Zapi
* Extrair metadados básicos:

  * Tipo de mídia
  * Identificador do arquivo
  * Link temporário ou referência
* Encaminhar para o pipeline interno do agente

Sem:

* Transcrição automática (se não existir hoje)
* Persistência adicional
* Preview complexo

---

### 3. Recebimento de Imagem

Adicionar lógica incremental para:

* Detectar mensagens de imagem
* Extrair:

  * Identificador
  * Link temporário
  * Metadados básicos
* Registrar internamente como mensagem recebida do tipo imagem

Sem:

* Processamento de imagem
* OCR
* Armazenamento permanente

---

### 4. Recebimento de Arquivo

Adicionar lógica incremental para:

* Detectar documentos/arquivos
* Extrair:

  * Tipo do arquivo
  * Nome (se disponível)
  * Tamanho
  * Link temporário
* Encaminhar para o fluxo interno

Sem:

* Download automático
* Indexação
* Persistência

---

### 5. Envio de Áudio

Adicionar endpoint ou função incremental para:

* Enviar áudio via Zapi
* Aceitar:

  * URL pública
  * Arquivo já disponível
* Retornar status de envio

Sem:

* Alterar envio de texto
* Unificar com outros tipos

---

### 6. Envio de Imagem

Adicionar fluxo incremental para:

* Enviar imagem via Zapi
* Aceitar:

  * URL pública
  * Arquivo já disponível
* Registrar como mensagem enviada

---

### 7. Envio de Arquivo

Adicionar fluxo incremental para:

* Enviar documentos/arquivos
* Aceitar:

  * URL pública
  * Arquivo já disponível
* Manter separado de texto
