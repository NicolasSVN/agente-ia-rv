# DPI adaptativo com PyMuPDF — simples e sem overhead
DPI_BY_CONTENT = {
    "text": 150,
    "mixed": 200,
    "table": 250,
    "infographic": 200,
    "image_only": 150,
}

def _page_to_image(self, page, dpi: int = 150) -> Image:
    zoom = dpi / 72.0
    matrix = fitz.Matrix(zoom, zoom)
    pix = page.get_pixmap(matrix=matrix)
    return Image.frombytes("RGB", [pix.width, pix.height], pix.samples)
```

A abordagem dos dois passes que discutimos antes era necessária porque com `pdf2image` você precisaria reprocessar o arquivo inteiro. Com PyMuPDF você tem o documento aberto em memória e muda só o zoom da página específica — custo praticamente zero.

**Terceiro ponto:** PyMuPDF consegue extrair texto nativo dos PDFs além de gerar imagens. Para PDFs com texto embutido (não escaneados), isso é mais rápido e mais barato do que mandar para o Vision. Não é para substituir o Vision — a extração estruturada de fatos e auto_tags que o Vision faz não tem equivalente — mas pode ser usado como pré-triagem: se a página tem texto nativo legível, o Vision recebe a imagem já com o texto disponível como hint, melhorando a extração.

## Como prevenir problemas similares no futuro

O padrão desse erro — "funciona em dev, falha em produção por dependência implícita de sistema" — é um dos mais comuns em deploy e um dos mais silenciosos, porque não aparece nos testes locais. Três proteções complementares:
```
Implemente um health check de dependências que roda na
inicialização da aplicação em produção:

# services/dependency_check.py

import logging
logger = logging.getLogger(__name__)

def check_critical_dependencies() -> dict:
    """
    Valida dependências críticas na inicialização.
    Falhas aqui são logadas como CRITICAL — nunca deixam
    a aplicação iniciar silenciosamente com recursos quebrados.
    """
    results = {}

    # PyMuPDF
    try:
        import fitz
        results["pymupdf"] = {"ok": True, "version": fitz.version[0]}
    except ImportError as e:
        results["pymupdf"] = {"ok": False, "error": str(e)}
        logger.critical("[STARTUP] PyMuPDF não disponível — upload de PDF vai falhar")

    # OpenAI
    try:
        from core.config import settings
        results["openai_key"] = {
            "ok": bool(settings.OPENAI_API_KEY),
            "configured": bool(settings.OPENAI_API_KEY)
        }
        if not settings.OPENAI_API_KEY:
            logger.critical("[STARTUP] OPENAI_API_KEY não configurada")
    except Exception as e:
        results["openai_key"] = {"ok": False, "error": str(e)}

    # pgvector
    try:
        import pgvector
        results["pgvector"] = {"ok": True}
    except ImportError as e:
        results["pgvector"] = {"ok": False, "error": str(e)}
        logger.critical("[STARTUP] pgvector não disponível — busca semântica vai falhar")

    # python-magic (validação de MIME)
    try:
        import magic
        results["python_magic"] = {"ok": True}
    except ImportError as e:
        results["python_magic"] = {"ok": False, "error": str(e)}
        logger.warning("[STARTUP] python-magic não disponível — validação MIME degradada")

    # Resumo
    failed = [k for k, v in results.items() if not v["ok"]]
    if failed:
        logger.critical(f"[STARTUP] {len(failed)} dependência(s) crítica(s) ausente(s): {failed}")
    else:
        logger.info(f"[STARTUP] Todas as dependências validadas: {list(results.keys())}")

    return results

# Chame no startup da aplicação (main.py ou app factory):
# @app.on_event("startup")
# async def startup_event():
#     check_critical_dependencies()
```

Isso transforma um erro silencioso que só aparece quando o primeiro usuário tenta fazer upload em um erro explícito que aparece nos logs imediatamente após o deploy — quando você ainda está monitorando.

A segunda proteção é adicionar um endpoint de health check que o Replit ou qualquer sistema de monitoramento pode pingar:
```
Crie um endpoint GET /api/health/detailed que retorne:

{
  "status": "ok" | "degraded" | "critical",
  "dependencies": {
    "database": "ok",
    "vector_store": "ok",
    "pdf_processing": "ok",
    "openai": "ok",
    "zapi": "ok"
  },
  "version": "...",
  "uptime_seconds": 123
}

Se qualquer dependência crítica falhar, retorne HTTP 503.
Isso permite detectar problemas antes que usuários os encontrem.
