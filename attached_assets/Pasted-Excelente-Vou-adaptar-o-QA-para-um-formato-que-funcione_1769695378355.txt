# QA Técnico para IA Dev (Replit) + Revisão Arquitetural

## Objetivo Técnico

Garantir que o sistema:

* Seja modular (Produto ≠ Documento ≠ Chunk)
* Tenha governança de versões
* Garanta freshness de dados
* Evite respostas com dados errados
* Seja observável (logs, métricas, auditoria)
* Tenha UX que reflita o modelo mental do broker
* Tenha RAG confiável para dados financeiros

---

## 1. Contratos de Dados (Data Contracts)

### Backend (Obrigatório)

Validar que existem entidades separadas e normalizadas:

Produto
Material
Bloco de Conteúdo
Versão
Fonte (PDF, Texto manual, Script, Tabela)

Critérios técnicos:

* Produto não depende de PDF
* Material referencia Produto
* Blocos têm tipo (text, table, chart, script)
* Versões são imutáveis
* Apenas uma versão ativa por bloco
* Metadados obrigatórios: vigência, confiança, status, updated_at, updated_by

Falha crítica:

* Atualizar conteúdo sobrescrevendo versão antiga

---

## 2. Pipeline de Ingestão (Robustez)

### Backend

Validar:

* Detecção de tipo por página
* Roteamento correto (texto, tabela, gráfico)
* Tabelas nunca chunkadas
* Gráficos sempre marcados pending_review
* Confidence score persistido
* status por item

Testes de falha:

* PDF com tabela grande
* PDF com imagem sem texto
* PDF híbrido (slides)

Falha crítica:

* Tabela fragmentada
* Gráfico auto-aprovado

---

## 3. Aprovação Híbrida (Controle Humano)

### Frontend

Validar:

* Revisão mostra só itens críticos
* Preview visual confiável
* Editor de tabela funcional
* Editor de descrição de gráfico
* Ação “aprovar tudo” respeita regras
* Nada crítico entra como approved sem revisão

Falha crítica:

* Item crítico entrando como approved automaticamente

---

## 4. Indexação & Freshness

### Backend + RAG

Validar:

* Só status=approved é indexado
* Reindex automático ao publicar nova versão
* Versão antiga removida do índice
* Filtros por vigência no retrieval
* Nenhum chunk expirado retornado

Testes:

* Atualizar taxa → buscar → garantir novo valor

Falha crítica:

* IA usando chunk antigo

---

## 5. Retrieval & Re-ranking

### RAG Layer

Validar:

* Filtros de metadata funcionando
* Re-ranking por tipo de pergunta
* Threshold mínimo de similaridade
* Top-K → Top-N reduzido
* Context window não poluído

Testes:

* Pergunta numérica → tabela priorizada
* Pergunta conceitual → texto priorizado

Falha crítica:

* Texto sendo usado para responder número

---

## 6. Scripts WhatsApp (Conteúdo Vivo)

### Front + Backend

Validar:

* Scripts são blocos independentes
* Versionamento ativo
* IA consegue recuperar script
* Scripts vinculados a produto
* Copiar/colar rápido

Falha crítica:

* Broker não conseguir atualizar script rápido

---

## 7. Transferência Humana (Fail-safe)

### Orquestração

Validar:

* Threshold de confiança
* Fallback externo
* Escalação humana automática
* Logs estruturados

Testes:

* Query sem base → humano
* Query fora escopo → humano

Falha crítica:

* IA inventar resposta

---

## 8. Observabilidade & Auditoria

### Plataforma

Validar:

* Logs de ingestão por documento
* Logs de retrieval por query
* Logs de versão usada
* Histórico de alterações
* Métrica de human_transfer

Falha crítica:

* Não saber qual fonte gerou resposta

---

## 9. UX Arquitetural (Produto)

### Frontend

Validar:

* Produto como hub central
* Navegação por Produto > Materiais > Blocos
* Atualização sem PDF
* Feedback visual de versão ativa
* Badges de expiração

Falha crítica:

* Broker ter que subir PDF para mudar taxa

---

## 10. Testes Automatizáveis para IA Dev

Sugestão para Replit:

Criar testes automatizados de:

* Ingestão de PDF com tabela
* Substituição de versão
* Retrieval com filtro de vigência
* Reindex ao aprovar
* Escalação humana

---

## Definition of Ready (para features novas)

Nenhuma feature nova entra se:

* Não respeitar modelo Produto > Bloco > Versão
* Não tiver metadados
* Não tiver rollback
* Não passar nos testes de freshness

---

## Arquitetura Red Flags (para você e para mim)

Se você vir qualquer um desses no código, é alerta:

* Atualização que sobrescreve registros
* Reindex manual
* Chunk sem referência a versão
* Retrieval sem filtro de status
* Frontend editando direto chunk
* PDFs como única fonte de verdade

---
