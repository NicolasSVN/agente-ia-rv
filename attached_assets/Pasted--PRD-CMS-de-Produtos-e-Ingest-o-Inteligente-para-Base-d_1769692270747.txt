# PRD — CMS de Produtos e Ingestão Inteligente para Base de Conhecimento

## Objetivo

Criar uma experiência simples para brokers e times comerciais gerenciarem materiais de produtos (one-pages, apresentações, scripts, taxas, etc.), garantindo que a IA sempre utilize a versão mais atual, estruturada e semanticamente rica.

O sistema deve se comportar como um **CMS de Produto**, não como um repositório técnico de PDFs.

---

## Modelo de Entidades

Produto
→ Material
→ Bloco de Conteúdo
→ Versão
→ Fonte

### Produto

Representa um produto financeiro específico.
Ex: XPTO Crédito Estruturado, Fundo ABC, COE XYZ

Campos:

* Nome do produto
* Gestora / Corretora
* Categoria
* Status (Ativo / Inativo)

---

### Material

Agrupa conteúdos por finalidade.

Tipos de material:

* One-page
* Apresentação completa
* Argumentos comerciais
* Atualização de taxas
* Scripts de WhatsApp
* FAQs
* Regulatório

Campos:

* Tipo
* Descrição
* Última atualização

---

### Bloco de Conteúdo

Menor unidade semântica indexável.

Exemplos:

* Tabela de taxas
* Texto de pitch
* Bullet points de diferenciais
* Script curto
* Descrição de gráfico

Campos:

* Tipo de bloco (Texto, Tabela, Script, Gráfico, Imagem)
* Conteúdo estruturado
* Tags semânticas

---

### Versão

Controle automático de histórico.

Campos:

* Número da versão
* Data
* Autor
* Motivo da alteração

---

### Fonte

Origem do conteúdo.

Tipos:

* PDF
* Texto manual
* Upload de planilha
* Geração por IA

## Fluxos Críticos

### Fluxo 1 — Atualização de taxa

1. Broker entra no produto
2. Abre One-page
3. Edita apenas o bloco de Tabela de Taxas
4. Salva
5. Sistema cria nova versão
6. IA passa a usar automaticamente

---

### Fluxo 2 — Novo texto comercial curto

1. Broker entra no produto
2. Aba Scripts
3. Novo Script
4. Cola texto
5. Salva
6. Script já é indexado

---

## Estratégia de Ingestão Inteligente

PDFs grandes devem ser tratados como:

* Fonte inicial
* Não como unidade principal

Pipeline ideal:

PDF → OCR + Layout → Quebra por slide → Classificação por tipo → Conversão em blocos estruturados → Revisão opcional → Indexação

---

## Princípios de UX

* Atualizar bloco, não PDF inteiro
* Pensar em produto, não arquivo
* Versionamento invisível
* Busca sempre retorna versão mais atual

---

## Benefícios

* Menos retrabalho
* Mais precisão semântica
* Atualizações rápidas
* Conteúdo pronto para IA, não só para humanos

---

## Métricas de Sucesso

* Tempo médio para atualizar taxa
* % de respostas da IA usando versão mais recente
* Redução de reprocessamento de PDFs
* Adoção do editor de blocos
