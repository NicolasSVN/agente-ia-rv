## Introdução

Criar um chatbot hoje é trivial. Qualquer desenvolvedor júnior consegue conectar uma API de LLM e receber respostas. Mas criar um sistema que **pensa**, **consulta conhecimento**, **toma decisões** e **responde com contexto auditável**? Isso é arquitetura cognitiva.

A maioria dos tutoriais na internet mostra apenas:

- Uma pergunta entra
- Uma resposta sai
- Zero memória
- Zero conhecimento externo
- Zero capacidade de decisão

**Neste laboratório prático, vamos além.**

Você vai construir um chatbot agêntico capaz de:

- Manter memória de conversa
- Consultar uma base de conhecimento (RAG)
- Decidir quando buscar informação externa
- Responder de forma contextual e explicável

## A Evolução Natural: Do Chat Burro ao Agente Inteligente

**Nível 1: Chatbot Tradicional**

```
Usuário: "Quem ganhou a Copa de 1970?"
Bot: "Não sei, isso não está no meu treinamento."
```

**Nível 2: Chatbot com Memória**

```
Usuário: "Oi, meu nome é João"
Bot: "Olá João!"
Usuário: "Qual meu nome?"
Bot: "João"
```

Progresso, mas ainda limitado ao que o modelo “lembra” da conversa.

**Nível 3: IA Agêntica com RAG (o que vamos construir)**

```
Usuário: "Quem ganhou a Copa de 1970?"
Agente: [Detecta necessidade de conhecimento]
        [Busca na base: "Copa 1970 Brasil Itália 4-1"]
        [Contextualiza com LLM]
Bot: "O Brasil venceu a Copa de 1970, derrotando a Itália
      por 4 a 1 na final realizada no México."
```

## Passo 3: Sistema RAG (Retrieval-Augmented Generation)

`rag.py` **:**

python

```
from openai import OpenAI
import faiss
import numpy as np
import os
from dotenv import load_dotenv
```

```
load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))class RAG:
    """Sistema de Recuperação Aumentada por Geração"""

    def __init__(self, texts):
        self.texts = texts
        self.index = self._create_index(texts)

    def _embed(self, text):
        """Gera embedding para um texto"""
        response = client.embeddings.create(
            model="text-embedding-3-small",
            input=text
        )
        return np.array(response.data[0].embedding)

    def _create_index(self, texts):
        """Cria índice vetorial FAISS"""
        print("🔄 Criando índice de embeddings...")
        embeddings = [self._embed(t) for t in texts]
        dimension = len(embeddings[0])
        index = faiss.IndexFlatL2(dimension)
        index.add(np.array(embeddings))
        print(f"✅ Índice criado com {len(texts)} documentos")
        return index

    def search(self, query, k=2):
        """Busca os k documentos mais similares"""
        query_embedding = self._embed(query)
        distances, indices = self.index.search(
            np.array([query_embedding]), k
        )
        results = [self.texts[i] for i in indices[0]]
        return results
```

**O que acontece aqui:**

- Cada frase da base vira um vetor matemático (embedding)
- FAISS organiza esses vetores para busca rápida
- Quando você pergunta algo, a pergunta vira vetor
- O sistema encontra os vetores mais próximos (semanticamente similares)

```
class OrchestratorAgent:
    """Agente orquestrador que decide quando usar RAG"""

    def __init__(self, rag):
        self.rag = rag

    def decide(self, question):
        """
        Decide se a pergunta precisa consultar a base de conhecimento.
        Lógica simples baseada em palavras-chave.
        """
        keywords = [\
            "final", "copa", "quem", "quando",\
            "onde", "venceu", "ganhou", "Brasil",\
            "1970", "México"\
        ]
        needs_rag = any(k in question.lower() for k in keywords)
        return needs_rag

    def retrieve_context(self, question):
        """Recupera contexto relevante da base"""
        return self.rag.search(question, k=3)
```

**Nota importante:** Esta lógica de decisão é deliberadamente simples. Em sistemas reais, você pode usar:

- Um segundo LLM para classificar intenções
- Análise de sentimento
- Classificadores treinados

## Passo 5: Interface Streamlit

`app.py` **:**

```
import streamlit as st
from openai import OpenAI
from rag import RAG
from agents import OrchestratorAgent
import os
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
# Configuração da página
st.set_page_config(
    page_title="Chat IA Agêntica",
    page_icon="🤖",
    layout="wide"
)
st.title("🤖 Chat com IA Agêntica + RAG")
st.caption("Um assistente que consulta conhecimento antes de responder")
# Inicializar memória da conversa
if "messages" not in st.session_state:
    st.session_state.messages = []
# Carregar base de conhecimento
@st.cache_resource
def load_rag():
    with open("data/conhecimento.txt") as f:
        docs = [line.strip() for line in f if line.strip()]
    return RAG(docs)
rag = load_rag()
agent = OrchestratorAgent(rag)
# Exibir histórico de conversa
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])
        if "context" in msg:
            with st.expander("📚 Contexto usado"):
                st.info("\n\n".join(msg["context"]))
# Input do usuário
question = st.chat_input("Digite sua pergunta...")
if question:
    # Adicionar pergunta ao histórico
    st.session_state.messages.append(
        {"role": "user", "content": question}
    )

    with st.chat_message("user"):
        st.markdown(question)

    # Decisão do agente
    context = []
    if agent.decide(question):
        with st.spinner("🔍 Consultando base de conhecimento..."):
            retrieved = agent.retrieve_context(question)
            context = retrieved

    # Construir prompt para o LLM
    if context:
        prompt = f"""Você é um assistente preciso que responde com base em fatos.
Contexto da base de conhecimento:
{chr(10).join(context)}
Pergunta do usuário:
{question}
Responda de forma clara e direta usando APENAS as informações do contexto.
Se o contexto não contiver a resposta, diga que não tem essa informação."""
    else:
        prompt = question

    # Gerar resposta
    with st.spinner("💭 Processando resposta..."):
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.3
        )

    answer = response.choices[0].message.content

    # Adicionar resposta ao histórico
    message_data = {"role": "assistant", "content": answer}
    if context:
        message_data["context"] = context

    st.session_state.messages.append(message_data)

    # Exibir resposta
    with st.chat_message("assistant"):
        st.markdown(answer)
        if context:
            with st.expander("📚 Contexto usado"):
                st.info("\n\n".join(context))
```

Observe que o agente adotou uma estratégia híbrida: quando o contexto recuperado não continha informação suficiente, o sistema recorreu ao conhecimento base do LLM. Esse comportamento demonstra a flexibilidade da arquitetura agêntica, mas também revela um ponto de atenção — em cenários onde precisão factual é crítica, pode ser necessário implementar validações adicionais ou restringir o modelo a responder exclusivamente com base no contexto fornecido.

## O Que Torna Isso IA Agêntica (de Verdade)?

## ✅ Capacidade de Decisão

O sistema **decide** quando buscar conhecimento externo, não faz isso cegamente.

## ✅ Prevenção de Alucinações

O LLM recebe contexto factual antes de responder, reduzindo invenções.

## ✅ Auditabilidade

Você pode ver exatamente quais documentos foram usados para gerar cada resposta.

## Armadilhas Comuns e Soluções

## ❌ Problema: Respostas genéricas mesmo com RAG

**Solução:** Ajustar o prompt para forçar uso do contexto:

```
prompt = """IMPORTANTE: Use APENAS informações do contexto abaixo.
Se não estiver no contexto, diga "Não tenho essa informação na base."
```

```
Contexto: {context}
Pergunta: {question}"""
```

## ❌ Problema: Busca retorna documentos irrelevantes

**Solução:** Aumentar granularidade dos chunks:

```
# Em vez de frases completas, use parágrafos temáticos
docs = split_into_paragraphs(text)
```

## ❌ Problema: Sistema lento

**Solução:** Cache de embeddings:

```
@st.cache_data
def get_embedding(text):
    return client.embeddings.create(...)
```

## Conclusão: O Que Você Construiu

Este não é um chatbot. É um **sistema cognitivo simples** com:

- **Percepção**: Entende a pergunta
- **Memória**: Consulta conhecimento externo
- **Raciocínio**: Usa LLM para processar contexto
- **Decisão**: Escolhe quando usar cada componente

A partir daqui, você não está mais ensinando “como usar IA”. Você está ensinando **arquitetura cognitiva aplicada**.
